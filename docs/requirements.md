# Бизнес-требования

## Цель
Автоматизировать публикацию товаров-забегов в WooCommerce на основе данных из Google Sheets, включая генерацию описаний и переводы, чтобы снизить ручной труд и ускорить выпуск контента.

## Источники данных
- Google Sheets: основная таблица с данными о забегах.
- Внешние источники: сайты организаторов и регламенты (PDF/HTML) для извлечения текста.

## Ключевые сценарии
- Система обрабатывает строки со статусами `Revised (incomplete)` и `Revised (complete)`.
- Для `Revised (incomplete)` генерируются поля `ORG INFO`, `SUMMARY`, `BENEFITS`, `FAQ` (EN/PT), затем публикуется карточка события, сохраняется baseline hash страницы `WEBSITE`, статус меняется на `Published (incomplete)`.
- Для `Published (incomplete)` ежедневно выполняется мониторинг изменений `WEBSITE` по сравнению hash (с нормализацией HTML); при изменениях отправляется уведомление в Telegram.
- Для `Revised (complete)` генерируется контент через OpenAI (описание, преимущества, информация об организаторе, FAQ), после чего данные обновляются в WordPress по `WP PRODUCT ID EN/PT`, а при отсутствии ID создаются новые товары.
- Опционально генерируется изображение и загружается в WordPress.
- Создается или обновляется variable-продукт в WooCommerce, настраиваются категории, атрибуты и вариации.
- Вариации синхронизируются по ID (`WP VARIATION ID EN`, `WP VARIATION ID PT`): существующие обновляются, новые создаются, отсутствующие в таблице удаляются из WooCommerce.
- Создается или обновляется португальский перевод товара через WPML API.
- В Google Sheets обновляется статус строки и дополнительные поля результата.

## Выходные данные
- Товары WooCommerce с вариациями и атрибутами.
- Переведенные товары (PT) с привязкой к оригиналу.
- Обновления в Google Sheets (статус, дополнительные поля, `WP PRODUCT ID EN`, `WP PRODUCT ID PT`, `WP VARIATION ID EN`, `WP VARIATION ID PT`, hash-поля мониторинга).
- Уведомления в Telegram о различиях страницы `WEBSITE`.

## Ограничения и допущения
- Все секреты и настройки берутся из переменных окружения.
- Модели OpenAI, уровень размышления, температура и пути к системным промптам задаются через `.env`.
- Доступ к Google Sheets обеспечивается через сервисный аккаунт.
- WordPress должен иметь активные WooCommerce, WPML и JWT Auth плагины.

## Нефункциональные требования
- Работа в Docker без локального окружения.
- Повторные попытки при временных сетевых ошибках к Google Sheets и WooCommerce.
- Логирование ключевых этапов обработки.
